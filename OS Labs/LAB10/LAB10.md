3.  Виконайте наступні практичні завдання у терміналі наступні дії:

- створіть трьох нових користувачів;  
Команди:  
useradd user1  
useradd user2  
useradd user3  
Пояснення: створюються нові користувачі.

- створіть нову групу користувачів...  
Команди:  
groupadd team  
usermod -aG team user1  
usermod -aG team user2  
Пояснення: створюється група та додаються користувачі.

- створіть новий файл...  
Команди:  
touch script.sh  
chmod 700 script.sh  
Пояснення: файл створено і доступний тільки власнику.

- для користувачів групи...  
Команди:  
chmod 750 script.sh  
Пояснення: група має read+execute.

- для інших користувачів...  
Команди:  
chmod 750 script.sh  
Пояснення: інші не мають доступу.

- подібні дії для директорій...  
Команди:  
mkdir dir_all && chmod 777 dir_all  
mkdir dir_owner && chmod 700 dir_owner  
mkdir dir_group && chmod 750 dir_group  
Пояснення: створені директорії з різними правами.

- emptyfile...  
Команди:  
touch emptyfile  
chmod 000 emptyfile  
chmod 4 emptyfile  
chmod 44 emptyfile  
Пояснення: перевірка роботи chmod.

- каталог...  
Команди:  
mkdir shared  
chmod 2770 shared  
chmod +t shared  
Пояснення: setgid + sticky.

- посилання...  
Команди:  
touch file1  
ln file1 hardlink  
ln -s file1 symlink  
Пояснення: створені посилання.

**Контрольні запитання:**

## Контрольні запитання та відповіді

**1. Наведіть приклади зміни прав доступу символічним методом (Symbolic Method)?**  
chmod u+x file

**2. Наведіть приклади зміни прав доступу числовим методом (numeric method, octal method)?**  
chmod 755 file

**3. Яке призначення команди umask?**  
Команда umask задає права доступу за замовчуванням для нових файлів і директорій.

**4. Порівняйте жорсткі та символічні посилання?**  
Жорстке посилання вказує на той самий inode файлу, а символічне є окремим файлом-посиланням.

**5. Чи можна виконати файл, для якого є права на виконання, але не встановлені права на читання (--x)? Поясніть.**  
Так, якщо встановлено право виконання, файл можна виконати навіть без права читання.

**6. Якщо ми змінюємо права доступу та дозволи в поточній сесії чи будуть вони збережені в наступній?**  
Так, права доступу зберігаються, оскільки записуються у файловій системі.

**7. Чи є якийсь шаблон, яким система користується щодо прав та доступів при створенні нових файлів. Як можна змінити права дозволу за замовчуванням?**  
Так, використовується шаблон umask, який можна змінити командою umask.

**8. Яким чином можна створити жорстке посилання? В яких ситуаціях їх доцільно використовувати?**  
Жорстке посилання створюється командою ln file link і використовується для створення альтернативного імені файлу.

**9. Яким чином можна створити символічне посилання? В яких ситуаціях їх доцільно використовувати?**  
Символічне посилання створюється командою ln -s file link і використовується як ярлик до файлу.

**10. Уявіть, що програмі потрібно створити одноразовий тимчасовий файл, який більше ніколи не знадобиться після закриття програми. Який правильний каталог для створення цього файлу?**  
/tmp

**11. Є файл оригінал та для нього створено два посилання - символічне та жорстке. Що відбудеться з іншими файлами, якщо видалити:**  
- файл оригінал - жорстке посилання залишиться працювати, символічне перестане  
- символічне посилання - нічого не зміниться  
- жорстке посилання - файл залишиться, якщо існує інше посилання
