**Тема: “Аналіз UX/UI дизайну предметів та інтерфейсів у повсякденному житті”**

**Мета роботи:**   
1\. Закріпити розуміння різниці між UX та UI дизайном.  
2\. Навчитися аналізувати інтерфейси реальних об’єктів та цифрових продуктів.  
3\. Розвинути критичне мислення щодо ефективності та зручності дизайну.

**Матеріальне забезпечення занять:**  
1\. Персональний комп'ютер, смартфон, доступ до мережі Інтернет, папір та ручка (або графічний редактор)  
2\. Будь-який побутовий прилад (для аналізу)

**Короткі теоретичні відомості:**  
Дизайн — це процес проєктування рішення проблеми.  
Інтерфейс — місце взаємодії двох систем (людини та машини).  
UI (User Interface) — "Як це виглядає". Кольори, шрифти, форми кнопок, матеріали.  
UX (User Experience) — "Як це працює". Логіка, зручність, швидкість досягнення мети, реакція системи, відсутність помилок.  
Афорданс (Affordance) — властивість об'єкта, яка підказує, як ним користуватися (кнопка випукла — її треба натиснути).

**Хід роботи**  
**Практичне завдання №1. "Реверс-інжиніринг" побутового приладу**

1. Оберіть складний побутовий прилад (пральна машина, мікрохвильовка, кавомашина, мультиварка) або ґаджет (смарт-годинник, фітнес-браслет).  
2. Зробіть фотографію (або підберіть) панелі керування приладом.  
3. Заповніть таблицю аналізу (наведіть мінімум по 3 пункти):  
   

| Складова | Опис елементів (Що ви бачите/відчуваєте?) | Яку проблему це вирішує? (Інженерна/Користувацька мета) |
| ----- | ----- | ----- |
| **UI (Вигляд)** | *Наприклад: Червоний колір кнопки "Стоп".* | *Привертає увагу у випадку небезпеки/помилки.* |
| **UI (Матеріал)** | *Наприклад: Прогумована ручка.* | *Запобігає ковзанню мокрих рук.* |
|  |  |  |
| **UX (Логіка)** | *Наприклад: Захист від випадкового натискання (треба тримати 3 сек).* | *Запобігає збою програми дітьми або випадковим дотиком.* |
| **UX (Розташування)** | *Наприклад: Екран знаходиться зверху.* | *Дозволяє бачити час, не нахиляючись.* |
|  |  |  |

   

   **Практичне завдання №2. \*Аналіз "Невидимого інтерфейсу" та Афордансу (фізичний світ)**

1. Розгляньте простий неелектронний інструмент (ложка, молоток, викрутка, канцелярський ніж).  
2. Опишіть його UI: матеріал, фактура, колір, зносостійкість.  
3. Опишіть його UX:  
* Як форма предмета підказує, як його тримати? (Це і є афорданс).  
* Як розподілено вагу предмета для зручності?  
* Що станеться, якщо змінити форму ручки на квадратну? (Опишіть наслідки для користувача)

**Практичне завдання №3. \*Аналіз "Невидимого інтерфейсу" та Афордансу (Цифровий світ)** 

1. Оберіть вебсайт або мобільний додаток і проаналізуйте:  
* UI: кольори, розташування кнопок, шрифти  
* UX: логіка навігації, сценарії використання, швидкість доступу до функцій  
2. Знайдіть у будь-якій комп'ютерній програмі або мобільному додатку елемент, який порушує принцип зрозумілості (поганий афорданс). Зробіть скріншот і опишіть, чому це помилка.  
* Приклад: Кнопка, яка не схожа на кнопку; текст, який виглядає як посилання, але не клікабельний.

**Практичне завдання №4. \*\*Проєктування інтерфейсу майбутнього** 

1. Запропонуйте концепцію нового виду інтерфейсу, якого ще не існує масово, або опишіть специфічний існуючий інтерфейс для конкретної ситуації (наприклад, інтерфейс для сапера, у якого зайняті руки, або для хірурга під час операції).  
2. Опишіть систему за схемою:  
* Назва технології.  
* Спосіб введення (Input): Як людина передає команду? (Думка, рух очей, мікроміміка, температура тіла тощо).  
* Спосіб виведення (Output/Feedback): Як система відповідає? (Вібрація, проєкція на сітківку, звук, пряма стимуляція нервів).  
* UI складова: Як це виглядає/відчувається естетично?  
* UX сценарій: Опишіть одну конкретну задачу, яку цей інтерфейс вирішує швидше за смартфон.


**Контрольні запитання**

1. Що спільного між графічним дизайном плаката та інтерфейсом програми?  
2. Чому інженери ПЗ повинні розуміти основи дизайну?  
3. Наведіть приклад предмета з добре продуманим UX.  
4. \*Ви розробляєте програму для заводу, де робітники працюють у товстих рукавицях. Які обмеження це накладає на UI та UX (розмір кнопок, тип екрану, жести)?  
5. \*Проаналізуйте інтерфейс командного рядка (CLI). У чому його поганий UX для новачка, і в чому чудовий UX для адміністратора?  
6. \*Що таке "інтуїтивно зрозумілий інтерфейс"?  
7. \*\*Уявіть ситуацію: користувачі постійно помиляються при вводі дати народження на сайті. Як це виправити засобами UI (візуал), а як — засобами UX (логіка)?  
8. \*\*Як зміна контексту використання впливає на UX? (Порівняйте використання навігатора в телефоні, коли ви сидите на дивані, і коли ви їдете за кермом на швидкості 100 км/год).
