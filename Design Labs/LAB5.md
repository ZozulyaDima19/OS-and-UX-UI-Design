| Term | Definition |
|-----|-----|
| Layout Grid | A system of intersecting vertical and horizontal lines used to organize, align, and structure elements in a design layout. |
| Column Grid | A grid made of vertical columns that helps structure content and align interface elements. |
| Row Grid | A grid composed of horizontal rows used to align elements across the vertical space of a layout. |
| Modular Grid | A grid formed by both rows and columns that creates modules for precise placement of elements. |
| Margin | The outer space between the edge of the screen and the content area. |
| Alignment | The positioning of elements so that their edges or centers line up along the same grid lines. |
| Consistency | Maintaining the same spacing, alignment, and structural rules across the entire interface. |
| Hierarchy | The visual importance of elements, often defined by their size, position, or width in the grid. |
| Responsive Design | A design approach that allows the layout to adapt to different screen sizes and devices. |
| Column Layout | A method of arranging content into vertical sections that follow the grid structure. |
| 8-Point System | A spacing system where margins, padding, and element sizes are multiples of 8 pixels. |

2. Дайте відповіді на наступні питання:

**Чим Task Flow відрізняється від User Flow?**  
Task Flow показує послідовність кроків для виконання однієї конкретної задачі без розгалужень. User Flow демонструє повний шлях користувача в системі та може містити різні сценарії, альтернативні дії і точки прийняття рішень.

**Чому вайрфрейми зазвичай виконуються у відтінках сірого?**  
Вайрфрейми виконують у відтінках сірого, щоб зосередитись на структурі інтерфейсу, розташуванні елементів і логіці взаємодії, не відволікаючись на кольори, графіку та деталі дизайну.

**Що таке «Grey Box Method» у прототипуванні?**  
Grey Box Method — це підхід до створення прототипів, у якому інтерфейс представляється у вигляді простих сірих блоків, що показують структуру і функціональні області майбутнього інтерфейсу без детального дизайну.

## **Контрольні запитання**

**Який елемент вайрфрейму вказує на те, що тут буде зображення?**  
У вайрфреймах зображення зазвичай позначаються прямокутником із діагональними лініями або іконкою зображення, що показує місце для майбутньої графіки.

**Навіщо використовувати реальні тексти замість "Lorem Ipsum" на етапі High-fi вайрфреймів?**  
Реальні тексти допомагають перевірити, як контент виглядатиме в інтерфейсі, чи вистачає місця для тексту та чи правильно працює структура ієрархії інформації.

**Що таке "Happy Path" у сценарії користувача?**  
Happy Path - це найпростіший і найуспішніший сценарій виконання задачі, коли користувач проходить всі кроки без помилок і досягає бажаного результату.

**Як Layout Grid допомагає у передачі макета розробнику?**  
Layout Grid задає чітку структуру колонок, відступів і вирівнювання елементів, що допомагає розробнику точно відтворити дизайн та забезпечити адаптивність інтерфейсу.

**Чому важливо проєктувати екран "Success State" (успіх), якщо дія відбувається на сервері (наприклад, надсилання листа)?**  
Екран Success State підтверджує користувачу, що дія була виконана успішно, зменшує невизначеність і покращує користувацький досвід.

## Conclusion

During this work, the main principles of creating wireframes and structuring user interfaces were studied. The concepts of User Flow and Task Flow were analyzed and their role in designing user interaction with a system was explained. Special attention was paid to the use of grayscale colors in wireframes, the Grey Box Method in prototyping, and the importance of designing clear user interaction scenarios such as the Happy Path and Success State.

In addition, the use of Layout Grid was examined as an important tool for organizing interface elements, maintaining alignment, consistent spacing, and visual consistency. Using a grid helps designers create well-structured layouts and makes it easier for developers to accurately implement the design.
