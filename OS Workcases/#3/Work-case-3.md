
# Work-case 3: Віртуальні машини та мережеві налаштування

У цьому завданні розглянемо процес клонування віртуальної машини, налаштування мережі між ними та обмін інформацією з основною ОС.

## Клонування віртуальної машини

Для клонування робочої ОС у VirtualBox (або іншому середовищі, наприклад VMWare) виконуємо наступні кроки:
1. Відкрити віртуальну машину, яку потрібно клонувати.
2. У VirtualBox натиснути **Machine → Clone**.
3. Обрати тип клонування: **Full clone** (повний клон) або **Linked clone** (зв'язаний клон, що використовує образ оригіналу).
4. Вказати ім'я для нового клону та підтвердити операцію.

Місце для скріну: ![Клонування VM](скрін_клону.png)

Для перенесення в інше віртуальне середовище потрібно експортувати віртуальну машину:
1. У VirtualBox вибрати **File → Export Appliance**.
2. Обрати віртуальну машину, яку потрібно експортувати.
3. Обрати формат, наприклад **.ova**.
4. Зберегти файл експорту. Цей файл можна імпортувати у будь-який сумісний віртуальний менеджер.

Місце для скріну: ![Експорт VM](скрін_експорт.png)

## Організація мережевих з'єднань у віртуальному середовищі

Існують кілька типів мережевих підключень:

**Трансляція мережевих адрес (NAT)** дозволяє віртуальній машині виходити в Інтернет через IP основної ОС. Внутрішні пристрої поза віртуальною машиною не бачать цю ОС.

**Мережевий міст (Bridged)** підключає віртуальну машину безпосередньо до фізичної мережі, даючи їй власний IP у тій же підмережі, що й основна ОС. Інші пристрої у локальній мережі бачать віртуальну машину як окремий комп'ютер.

**Віртуальний адаптер хоста (Host-only)** створює мережу, де віртуальна машина може спілкуватися тільки з основною ОС та іншими VM на цьому адаптері, без доступу в Інтернет.

**Внутрішня мережа (Internal Network)** дозволяє віртуальним машинам обмінюватися даними між собою, ізольовано від хосту та Інтернету.

## Налаштування мережі між робочою ОС та її клоном

Після клонування, щоб обидві ОС мали доступ до мережі та між собою, можна налаштувати NAT або Bridged для Інтернету та Host-only для локальної взаємодії.

У терміналі Linux базові команди для мережевих параметрів:
```bash
# Перевірка наявних інтерфейсів
ip addr

# Присвоєння статичного IP для локальної мережі
sudo ip addr add 192.168.56.101/24 dev enp0s8
sudo ip link set enp0s8 up

# Перевірка зв'язку з іншою VM
ping 192.168.56.102
```
Ці команди налаштовують локальні IP-адреси та дозволяють перевірити підключення між машинами.

Місце для скріну: ![Ping VM](ping_vm.png)

Обидві ОС мають вихід в Інтернет, тому можна відкрити браузер та переглянути відео на YouTube.

Для обміну повідомленнями у локальній мережі можна використовувати команду `ssh` або `netcat`:
```bash
# Відправлення повідомлення з однієї VM на іншу
nc 192.168.56.102 12345
```

## Спільна мережева папка

Для обох ОС можна налаштувати спільну папку через VirtualBox:
1. В налаштуваннях VM обрати **Shared Folders → Add Folder**.
2. Вказати шлях до папки на хості та обрати **Auto-mount**.
3. У VM доступ до папки буде через `/media/sf_<ім'я папки>`.

Скопіювати файли можна командами:
```bash
cp /media/sf_shared/file.txt ~/
cp /media/sf_shared/file.txt ~/Desktop/
```

## Обмін інформацією з основною ОС

Між основною ОС (Windows) та VM можна обмінюватися файлами через спільні папки або drag&drop (якщо ввімкнено Guest Additions/VMware Tools). Наприклад, аудіо-файл можна скопіювати на робочий стіл обох VM.

Зворотне копіювання з VM на основну ОС також відбувається через спільні папки або drag&drop. У терміналі Linux це можна зробити так:
```bash
cp ~/Desktop/document.txt /media/sf_shared/
```
Файл з’явиться у папці на хості.
