# Управління процесами та потоками в сучасних ОС Windows 10 та Windows 11

## Вступ

Сучасні операційні системи повинні ефективно керувати виконанням програм, розподіляти ресурси комп’ютера та забезпечувати стабільну роботу системи. Одним із ключових механізмів для досягнення цієї мети є **управління процесами та потоками**. Саме ці елементи відповідають за виконання програм, розподіл процесорного часу, використання пам’яті та взаємодію між різними частинами програмного забезпечення.

Операційні системи **Windows 10** та **Windows 11** використовують схожі механізми керування процесами та потоками, оскільки обидві базуються на **Windows NT Kernel**. Проте Windows 11 має певні покращення у плануванні задач, оптимізації використання процесора та інтеграції з новими типами процесорів.

---

## Поняття процесу

**Процес** - це виконувана програма, яка має власний адресний простір пам’яті, набір ресурсів та хоча б один потік виконання.

Кожен процес у Windows має:

- власний **ідентифікатор процесу (PID)**  
- **віртуальний адресний простір пам’яті**  
- дескриптори відкритих файлів  
- набір потоків  
- контекст виконання  

Процеси ізольовані один від одного, що забезпечує стабільність системи. Якщо один процес завершується аварійно, це не повинно вплинути на інші процеси.

### Основні стани процесу

У Windows процес може перебувати у декількох станах:

- **Створення (Created)** – процес створюється, але ще не виконується  
- **Готовий (Ready)** – процес готовий до виконання та очікує процесорного часу  
- **Виконання (Running)** – процес виконується на процесорі  
- **Очікування (Waiting)** – процес очікує на ресурс або подію  
- **Завершений (Terminated)** – процес завершив виконання  

---

## Поняття потоку

**Потік (Thread)** - це найменша одиниця виконання програми, яка планується операційною системою.

Один процес може містити **один або кілька потоків**, які виконуються паралельно. Потоки одного процесу мають спільний доступ до:

- пам’яті процесу  
- глобальних змінних  
- відкритих ресурсів  

Однак кожен потік має власні:

- регістри процесора  
- стек  
- лічильник команд  

Використання потоків дозволяє програмам працювати швидше та ефективніше використовувати багатоядерні процесори.

---

## Планування процесів і потоків

Планування - це механізм розподілу процесорного часу між процесами та потоками.

Windows використовує **пріоритетне витісняюче планування (preemptive priority scheduling)**. Це означає, що:

- кожен потік має **рівень пріоритету**
- потік з вищим пріоритетом отримує процесор раніше
- якщо з’являється потік з більшим пріоритетом, поточний потік може бути **витіснений**

Рівні пріоритету в Windows поділяються на:

- **Idle**
- **Below Normal**
- **Normal**
- **Above Normal**
- **High**
- **Real-time**

## Управління процесами в Windows

Для управління процесами Windows використовує кілька системних інструментів.

### Диспетчер завдань (Task Manager)

Диспетчер завдань дозволяє:

- переглядати активні процеси
- завершувати процеси
- відстежувати використання CPU, RAM, диска та мережі
- змінювати пріоритет процесів

![](https://www.lifewire.com/thmb/LGh8eIvqxll8X18cgRNPt6kEE8M=/1500x0/filters:no_upscale():max_bytes(150000):strip_icc()/task-manager-windows-11-980ae80d39694a14b75ddec1628d1924.png)

### Resource Monitor

Цей інструмент дозволяє детально аналізувати використання ресурсів системи.

Можна переглянути:

- активні процеси
- використання пам’яті
- навантаження на диск
- мережеву активність

![](https://www.top-password.com/blog/wp-content/uploads/2020/03/resource-monitor.png)

### Process Explorer

Це розширений інструмент від Microsoft Sysinternals, який дозволяє детально досліджувати процеси.

Можливості:

- перегляд дерева процесів
- аналіз відкритих файлів
- перегляд потоків процесу
- пошук DLL та дескрипторів

## Управління потоками

Windows надає розробникам API для створення та управління потоками.

Основні функції:

- **CreateThread()** — створення потоку  
- **SuspendThread()** — призупинення потоку  
- **ResumeThread()** — відновлення потоку  
- **TerminateThread()** — завершення потоку  

Для синхронізації потоків використовуються:

- **Mutex**
- **Semaphore**
- **Critical Section**
- **Event objects**

Ці механізми дозволяють уникати конфліктів доступу до спільних ресурсів.

## Порівняння Windows 10 та Windows 11

| Характеристика | Windows 10 | Windows 11 |
|---|---|---|
| Ядро системи | Windows NT | Windows NT |
| Планувальник процесів | Стандартний NT Scheduler | Оптимізований для нових CPU |
| Підтримка гібридних процесорів | Обмежена | Покращена (Intel Thread Director) |
| Оптимізація багатопоточності | Хороша | Покращена |
| Управління енергоспоживанням | Стандартне | Покращене |

Однією з головних відмінностей Windows 11 є покращена підтримка **гібридних процесорів**, де є продуктивні (P-cores) та енергоефективні (E-cores) ядра. Завдяки технології **Intel Thread Director** система може ефективніше розподіляти потоки між різними ядрами.

---

## Практичне значення управління процесами та потоками

Ефективне управління процесами та потоками дозволяє:

- підвищити продуктивність системи  
- забезпечити стабільність роботи ОС  
- оптимізувати використання процесора  
- реалізувати багатозадачність  

Саме завдяки цьому сучасні операційні системи можуть одночасно виконувати десятки або навіть сотні програм.

---

## Висновок

In conclusion, process and thread management plays a crucial role in modern operating systems such as Windows 10 and Windows 11. These mechanisms allow the operating system to efficiently allocate CPU time, manage system resources, and ensure stable multitasking. While both operating systems are based on the same Windows NT architecture, Windows 11 introduces several improvements in scheduling and optimization for modern processors. As hardware continues to evolve, effective process and thread management remains one of the most important components for maintaining high system performance and reliability.
